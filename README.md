# Polygon Agent

Describe a problem to your LLM agent (opencode, Claude, Copilot, Cursor).
It generates all 12 required files and uploads to Polygon. Zero manual steps.

## Setup

Add POLYGON_API_KEY and POLYGON_API_SECRET to .env

## Usage

Point your LLM at AGENTS.md and describe the problem.
The agent runs all 4 steps (scaffold, generate, check, upload) automatically.

## CLI Commands

  create  name [opts]    Scaffold a new problem folder
  check   name           Verify 12 files complete (shows Polygon ID if uploaded)
  upload  name [--force] Upload to Polygon, stores problem_id in state.json
  list                   List all local problems with stage and Polygon ID
  open    name           Open uploaded problem on Polygon in browser

create options: --title --author --legend --input --output
  --constraints --solution-idea --time-limit --memory-limit
  --tags --multitest --samples

## Required Files (12 total)

  statement/legend.tex     Problem story (LaTeX, 3-5 sentences)
  statement/input.tex      Input format + constraints
  statement/output.tex     Output format
  statement/notes.tex      Sample explanations
  statement/tutorial.tex   Editorial: Observations + Approach + Complexity
  validator.cpp            Strict input validator (testlib)
  checker.cpp              Output checker (testlib)
  generators/generator.cpp Test generator with full gen_* namespaces
  generators/script.txt    FreeMarker script: Polygon generates tests N+1..30
  solutions/acc.cpp        Main C++ solution (tag MA)
  solutions/java.java      Java solution (tag OK)
  solutions/brute.cpp      Brute-force/alternative solution (tag OK or TL)

## Test Structure

  Tests 1..N    Hand-made samples from spec.json (shown in statement)
  Tests N+1..30 Generated by Polygon via generators/script.txt

## spec.json fields

  name, title, author, legend, input, output, constraints, solution_idea
  time_limit_ms, memory_limit_mb, multitest, samples, tags
  brute_tag (OK or TL), statement_language

## ECPC 2026 Compliance

  Statement: clear English + I/O + Constraints + Samples
  Solutions: Main C++ (MA) + Java (OK) + Brute-force (OK or TL)
  Generator with usage instructions + Validator (both required)
  Tests hit MAX constraint values; edge cases + stress tests
  Tutorial mandatory: Key Observations + Approach + Complexity
  Tags set on Polygon. ICPC quality (not trivial, not obscure).
